  ## 1. 项目定义                                                                
                                                                                
  > **一套针对时间轴媒体的语义处理管道。它做的事情本质上只有一件：把原始媒体文  
  件，编译成一份携带完整语义标注的时间轴清单，再基于这份清单做任意后续决策。**  
                                                                                
  视频文件是输入，不是系统的核心资产。**时间轴清单（TimelineManifest）才是核心  
  资产。**                                                                      
                                                                                
  这不是一个剪辑工具——剪辑只是这份语义资产最显而易见的消费方式，但远非唯一。视  
  频剪辑是第一个应用，但架构的射程是一切时间轴媒体的语义化。

  ## 5. 2a 理解子阶段                                                           
                                                                                
  ### 5.1 执行方式                                                              
                                                                                
  - 固定两次调用：                                                              
    - **R1**：粗理解与候选识别                                                  
    - **R2**：精化理解与生成最终可消费结果                                      
                                                                                
  ### 5.2 R1 输出（中间态）                                                     
                                                                                
  - `purpose_rough`                                                             
  - `segments_rough[]`（草稿分块）                                              
  - `symbol_table_candidates[]`（易错词候选）                                   
                                                                                
  ### 5.3 R2 输出（写入 manifest）                                              
                                                                                
  - `purpose`                                                                   
  - `outline_blocks[]`                                                          
  - `cleaned_annotations[]`                                                     
                                                                                
  ### 5.4 输入约束                                                              
                                                                                
  - 输入源为 `annotations[] + goal`。                                           
  - Layer 1 原始文本不被覆盖，消歧通过 `cleaned_annotations[]` 旁路表达。       
                                                                                
  ---                                                                           
                                                                                
  ## 6. 2b 决策子阶段                                                           
                                                                                
  ### 6.1 输入（固定）                                                          
                                                                                
  - `comprehension.cleaned_annotations[]`                                       
  - `comprehension.purpose`                                                     
  - `comprehension.outline_blocks[]`（块 index 范围 + 块总结）                  
                                                                                
  代码层同时读取：                                                              
  - `annotations[]`（用于 silence 处理、长度/索引校验）                         
                                                                                
  ### 6.2 输出                                                                  
                                                                                
  - `keep_mask[]`（长度必须等于 `len(annotations)`）                            
  - `checklist_coverage[]`（MVP 预留）                                          
                                                                                
  ### 6.3 校验与重试                                                            
                                                                                
  - 必须校验 `keep_mask` 的长度、索引对齐、类型约束。                           
  - 失败时整体重试，最多 3 次。

  ## 7. 2c 审核子阶段（MVP 两阶段）                                             
                                                                                
  ### 7.1 Phase A（先交付）                                                     
                                                                                
  - 真实审核逻辑不启用。                                                        
  - 写入一条占位报告：`verdict="pass"`，用于维持 schema 兼容与流程一致性。      
                                                                                
  ### 7.2 Phase B（后续交付）                                                   
                                                                                
  - 再接入真实 LLM 审核逻辑（`pass/fix_decision/fix_checklist`）。              
  - 相关循环与预算策略不属于 MVP 本文范围。

  ## 8. 2d 人工子阶段（MVP）                                                    
                                                                                
  ### 8.1 仅保留手动勾选                                                        
                                                                                
  - 展示当前决策结果（可按 index 或按块辅助展示）。                             
  - 用户手动切换指定 index 的 keep/cut。                                        
  - 变更记录写入 `overrides` delta，不原地改写模型原始 `keep_mask`。            
                                                                                
  ### 8.2 合并与确认

  - 2d 覆盖的是 2b 决策输出。

  ## 10. 与 Layer 1 / Layer 3 的契约                                            
                                                                                
  - Layer 2 与 Layer 1 对齐依赖 `annotation.index` 稳定一致。                   
  - **智能层（含 2d）对执行层的交付物是定稿 `keep_mask`**（与 `annotations`     
  等长）；**不是** `edl[]`。                                                    
  - 识别层输出中带 **`index` 与时间段**（如 `t_start` /                         
  `t_end`）的序列，构成「时间–index 映射」；执行层用其与 `keep_mask`            
  合并后**在内部**得到连续时间上的 keep 区间，再**在执行层内部**合成 EDL        
  并驱动剪切。                                                                  
  - Layer 2 语义决策主坐标为 `index`；**时间换算与 EDL 合成**属于执行层职责。   
                                                                                
  ---                                                                           
                                                                                
  ## 11. 层间 JSON 解耦与 EDL 归属                                              
                                                                                
  - **解耦方式**：识别层、智能层、执行层之间通过**约定 JSON                     
  文件**读写交接（具体文件名与字段命名与工程一致时，以                          
  [AutoSmartCut-MVP.md](AutoSmartCut-MVP.md) 为准）。                           
  - **识别层输出 JSON**：提供带 `index`                                         
  的标注序列及每条对应的**时间边界**（以及 speech/silence                       
  等类型），即执行层所需的「时间–index 映射」来源。                             
  - **智能层输出 JSON**：提供与上述序列 **等长对齐** 的 **`keep_mask`**（经 2b  
  与 2d `overrides` 合并后的定稿）。                                            
  - **EDL**：由**执行层**读取上述两份 JSON 后，在**执行层内部**将 `keep_mask`   
  与时间–index 映射合并计算得到；**不作为 Layer 2                               
  的输出字段**，也不应在智能层落盘为对外契约的一部分。

  2.1 输入结构（两阶段通用）

  - 按句子索引与文本构造：

  3. 主要流程设计

  1. 输入构建
  从manifest["annotations"]抽取index和文本，构造JSON输入，第二阶段附加第一阶段输
  出数据。
  2. 调用LLM
  通过现有intelligence_llm.py调用接口，传入构造的Prompt和上下文，支持调用失败重
  试最多3次。
  3. 输出解析
  解析LLM返回的JSON文本，验证字段与结构，确保包含必需的主旨、分块、错词数据。
  4. 文本替换
  第二阶段完成后，利用corrections映射函数遍历逐句定位替换对应词（以索引和词内位
  置匹配），生成cleaned_annotations供后续2b使用。

  4. Prompt设计概要

  4.1 第一阶段 Prompt 设计要点

  - 明确指令：解析输入句子文本列表，输出包含粗主旨、内容主题分块、错词候选（含词
  序号和建议列表）三部分的严格JSON。
  - 禁止回复与JSON无关内容。
  - 举例演示结构，帮助模型理解格式。

  4.2 第二阶段 Prompt 设计要点

  - 任务说明：基于第一阶段结果，对主旨、分块和错词候选进行校验和确认，输出最终精
  准主旨、分块和唯一错词正确替换映射（含词序号）。
  - 输入包含原始文本和第一阶段结构化输出。
  - 明确输出为纯JSON，不输出全文，确保结构化和规则化。

  import json
  from typing import List, Dict, Any


  def extract_index_text(annotations: List[Dict[str, Any]]) -> List[Dict[str, 
  Any]]:
      """提取 index 和文本构成的简化列表"""
      return [{"index": ann["index"], "text": ann["content"]} for ann in
  annotations if ann["type"] == "speech"]


  def apply_corrections_to_annotations(
      annotations: List[Dict[str, Any]],
      corrections: List[Dict[str, Any]]
  ) -> List[Dict[str, Any]]:
      """逐句针对指定词（带词内序号）用修正词替换，生成新的消歧版annotations"""
      corrected = []
      for ann in annotations:
          if ann["type"] != "speech":
              corrected.append(ann)
              continue

          text = ann["content"]
          # 简单词拆分（按空格，替换时需更复杂分词可替换）
          words = text.split()
          ann_index = ann["index"]

          # 找出本句所有需要替换的词（按index匹配）
          replacements = [(c["original"][1], c["corrected"]) for c in
  corrections if c["index"] == ann_index]

          # 替换词
          for pos, new_word in replacements:
              if 0 <= pos < len(words):
                  words[pos] = new_word

          new_text = " ".join(words)
          new_ann = dict(ann)
          new_ann["content"] = new_text
          corrected.append(new_ann)

      return corrected

  请校验并优化文章主旨(purpose)、分块(outline_blocks)，
  并对每个错词候选选出唯一的修正词(corrections)，格式如下：
  {{
    "purpose": "...",
    "outline_blocks": [{{"start_index": 0, "end_index": 4, "topic": 
  "精准主题"}}, ...],
    "corrections": [{{"index": 3, "original": ["错词", 2], "corrected": 
  "修正词"}}, ...]
  }}
  请只输出JSON，不要多余说明。
  输入数据如下：
  {input_json}
  目标说明：
  {goal}
  """
      return prompt

  def parse_r2_output(llm_response: str) -> Dict[str, Any]:
      try:
          parsed = json.loads(llm_response)
          assert "purpose" in parsed
          assert "outline_blocks" in parsed
          assert "corrections" in parsed
          return parsed
      except Exception as e:
          raise ValueError(f"R2输出解析失败: {e}")

  def apply_corrections_to_annotations(
      annotations: List[Dict[str, Any]],
      corrections: List[Dict[str, Any]]
  ) -> List[Dict[str, Any]]:
      corrected = []
      for ann in annotations:
          if ann["type"] != "speech":
              corrected.append(ann)
              continue
          words = ann["content"].split()
          ann_index = ann["index"]
          # 找当前句相关替换
          replacements = [(c["original"][1], c["corrected"]) for c in
  corrections if c["index"] == ann_index]
          for pos, new_word in replacements:
              if 0 <= pos < len(words):
                  words[pos] = new_word
          new_ann = dict(ann)
          new_ann["content"] = " ".join(words)
          corrected.append(new_ann)
      return corrected

  def run_2a_comprehension(manifest: Dict[str, Any]) -> Dict[str, Any]:
      annotations = manifest["annotations"]
      goal = manifest.get("goal", "")

      input_r1 = extract_index_text_via_build(annotations)

      prompt1 = prompt_r1(input_r1, goal)
      raw_r1_output = intelligence_llm.call_llm_with_retry(prompt1)
      r1_result = parse_r1_output(raw_r1_output)

      input_r2 = {
          "annotations": input_r1,
          "outline_blocks_rough": r1_result["outline_blocks_rough"],
          "candidate_misrecognitions": r1_result["candidate_misrecognitions"],
      }
      prompt2 = prompt_r2(input_r2, goal)
      raw_r2_output = intelligence_llm.call_llm_with_retry(prompt2)
      r2_result = parse_r2_output(raw_r2_output)

      manifest["comprehension"] = r2_result
      manifest["cleaned_annotations"] = apply_corrections_to_annotations(
          annotations, r2_result.get("corrections", [])
      )

      return manifest
